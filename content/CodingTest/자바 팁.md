# tips

## record

``` java
// 기존 방식
static class Node {
    int id;
    int cost;
    public Node(int id, int cost) { this.id = id; this.cost = cost; }
}

// record 활용 (생성자, getter, equals, hashCode 자동 생성)
record Node(int id, int cost) implements Comparable<Node> {
    @Override
    public int compareTo(Node o) {
        return Integer.compare(this.cost, o.cost);
    }
}
```

## map


* **빈도수 계산 (`getOrDefault`):**

```java
Map<String, Integer> map = new HashMap<>();
map.put(key, map.getOrDefault(key, 0) + 1);
```

* **그래프 인접 리스트 초기화 (`computeIfAbsent`):**

```java
Map<Integer, List<Integer>> graph = new HashMap<>();
// 키가 없으면 ArrayList를 만들고, 있으면 기존 리스트에 바로 add
graph.computeIfAbsent(u, k -> new ArrayList<>()).add(v);
```

## int array <-> List<Integer>

* **`int[]` $\rightarrow$ `List<Integer>`**

``` java
int[] arr = {1, 2, 3};
List<Integer> list = Arrays.stream(arr).boxed().collect(Collectors.toList());
```

* **`List<Integer>` $\rightarrow$ `int[]`**

```java
List<Integer> list = List.of(1, 2, 3);
int[] arr = list.stream().mapToInt(Integer::intValue).toArray();
```

## 2차원 배열 정렬 람다식

``` java
int[][] intervals = {{1, 3}, {2, 5}, {4, 8}};

// 끝나는 시간(index 1) 기준 오름차순, 같다면 시작 시간(index 0) 기준 오름차순 정렬
Arrays.sort(intervals, (a, b) -> {
    if (a[1] == b[1]) return Integer.compare(a[0], b[0]);
    return Integer.compare(a[1], b[1]);
});
```

## 정렬

### 1. `Comparable<T>` : 객체의 '기본(자연스러운)' 정렬 기준 세우기

객체 자체가 "나는 이렇게 정렬되는 게 기본이야!"라고 정의할 때 사용합니다. 클래스 선언부에 `implements Comparable<T>`를 추가하고 `compareTo` 메서드를 오버라이드합니다.

* **리턴 값의 핵심 규칙:** * `음수` 또는 `0`: 자리 바꿈이 일어나지 않음 (현재 순서 유지)

    * `양수`: 자리 바꿈이 일어남 (순서가 뒤집힘)

    * **꿀팁:** 뺄셈(`this.value - o.value`)을 써도 되지만, 오버플로우 방지를 위해 `Integer.compare()`나 `Double.compare()`를 사용하는 것이 가장 안전합니다.

``` java
class Item implements Comparable<Item> {
    int price;
    String name;

    public Item(int price, String name) {
        this.price = price;
        this.name = name;
    }

    @Override
    public int compareTo(Item o) {
        // 1. 가격 기준 오름차순 정렬 (기본)
        return Integer.compare(this.price, o.price);
        // 만약 내림차순을 원한다면 순서를 바꿉니다.
        // return Integer.compare(o.price, this.price);
    }
}

// 사용 시: Arrays.sort(items); 또는 Collections.sort(itemList);
```

### 2. `Comparator<T>` (익명 클래스) : 그때그때 다른 기준 적용하기

객체에 `Comparable`이 구현되어 있지 않거나, 기본 정렬 기준을 무시하고 **새로운 기준(예: 이름순 정렬 등)으로 정렬하고 싶을 때** 사용합니다.

```java
// 익명 객체를 생성하여 compare 메서드 오버라이드
Arrays.sort(items, new Comparator<Item>() {
    @Override
    public int compare(Item o1, Item o2) {
        // 이름 기준 사전순 정렬 (String 클래스에 내장된 compareTo 활용)
        return o1.name.compareTo(o2.name);
    }
});
```

### 3. 람다식(Lambda)(Java 8+)

```java
// 단일 조건 정렬 (이름순)
Arrays.sort(items, (o1, o2) -> o1.name.compareTo(o2.name));

// 다중 조건 정렬 (1순위: 가격 오름차순, 2순위: 이름 내림차순)
Arrays.sort(items, (o1, o2) -> {
    if (o1.price == o2.price) {
        return o2.name.compareTo(o1.name); // 가격이 같으면 이름 내림차순
    }
    return Integer.compare(o1.price, o2.price); // 기본은 가격 오름차순
});
```

### 4. `Comparator` 유틸리티 메서드(Java 8+)

메서드 체이닝을 활용하여 영어를 읽듯 자연스럽게 정렬 기준을 작성하는 방법입니다.

```java
// 1. 가격 기준 오름차순
Arrays.sort(items, Comparator.comparingInt(item -> item.price));

// 2. 가격 기준 내림차순 (reversed 활용)
Arrays.sort(items, Comparator.comparingInt((Item item) -> item.price).reversed());

// 3. 다중 조건: 가격 오름차순 후, 가격이 같으면 이름 사전순 (thenComparing 활용)
Arrays.sort(items, Comparator.comparingInt((Item item) -> item.price)
                             .thenComparing(item -> item.name));
```

  

## stream

### 1. 컬렉션(Collection)과 배열(Array) 상호 변환 (가장 많이 쓰임!)

가장 귀찮고 빈번하게 발생하는 타입 변환을 한 줄로 해결할 수 있습니다.

**`int[]` ↔ `List<Integer>` 변환**

```java
int[] arr = {1, 2, 3, 4, 5};

// 1. int[] -> List<Integer>
List<Integer> list = Arrays.stream(arr).boxed().collect(Collectors.toList());

// 2. List<Integer> -> int[]
int[] newArr = list.stream().mapToInt(Integer::intValue).toArray();
```

**`String[]` ↔ `int[]` 변환** (입력값 파싱할 때 치트키)

```java
String[] strArr = {"10", "20", "30"};

// String 배열을 int 배열로 단숨에 변환
int[] intArr = Arrays.stream(strArr).mapToInt(Integer::parseInt).toArray();
```

### 2. 수학 연산 및 통계 구하기

반복문을 돌며 합계나 최댓값을 구할 필요가 없습니다.

```java
int[] arr = {1, 5, 3, 2, 4};

// 1. 합계, 최댓값, 최솟값, 평균
int sum = Arrays.stream(arr).sum();
int max = Arrays.stream(arr).max().getAsInt(); // OptionalInt 반환이므로 getAsInt() 필요
int min = Arrays.stream(arr).min().getAsInt();
double avg = Arrays.stream(arr).average().getAsDouble();

// 2. 배열 내 짝수만 골라서 더하기 (filter 체이닝)
int evenSum = Arrays.stream(arr).filter(n -> n % 2 == 0).sum();
```

### 3. 문자열 처리와 출력 (문자열 붙이기, 분리하기)

정답을 특정 포맷의 문자열로 출력해야 할 때 매우 강력합니다.

**배열의 요소들을 하나의 문자열로 결합**

```java
String[] words = {"Java", "Backend", "Developer"};

// 1. 단순히 붙이기 -> "JavaBackendDeveloper"
String res1 = Arrays.stream(words).collect(Collectors.joining());


// 2. 구분자 넣어서 붙이기 -> "Java, Backend, Developer"
String res2 = Arrays.stream(words).collect(Collectors.joining(", "));

// 3. 접두사, 접미사까지 넣기 -> "[Java, Backend, Developer]"
String res3 = Arrays.stream(words).collect(Collectors.joining(", ", "[", "]"));
```

  

**문자열을 잘라서 정렬하고 다시 붙이기**

```java
String str = "dcba";

// "abcd" 로 알파벳 순 정렬
String sortedStr = str.chars() // IntStream 반환 (아스키코드)
                      .sorted()
                      .collect(StringBuilder::new, StringBuilder::appendCodePoint, StringBuilder::append)
                      .toString();
```

### 4. `IntStream.range()`를 활용한 인덱스 생성

일반적인 `for (int i = 0; i < N; i++)`를 대체하는 용도로 쓰입니다. 주로 원본 배열을 건드리지 않고 인덱스를 조작할 때 사용합니다.

```java
String[] names = {"A", "B", "C"};
int[] scores = {80, 95, 70};

// 점수가 90점 이상인 사람의 '인덱스'만 추출
int[] topScorerIndices = IntStream.range(0, scores.length)
                                  .filter(i -> scores[i] >= 90)
                                  .toArray();
```
